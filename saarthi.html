The challenge

USMLE Sarthi's existing website suffered from cluttered navigation, outdated visual hierarchy, and a steep learning curve for international medical graduates preparing for US residency. Users struggled to find essential resources, and the manual CV creation process for ERAS was time-consuming and error-prone.

Design approach

Adopting a user-centered design methodology, we conducted extensive user interviews with IMGs, medical students, and residency advisors. Key insights revealed that users needed clear content organization, faster access to study materials, and guided assistance for complex ERAS requirements. The redesign focused on intuitive information architecture, reduced cognitive load, and AI-assisted personalization.

🔧 Technical challenges & solutions

  • ▹ WebSocket stability: Implemented automatic reconnection with exponential backoff, maintaining 99.9% uptime.
  • ▹ Redis optimization: LRU caching strategy reduced memory footprint by 40%.
  • ▹ ERAS compliance: Dynamic template engine adapts to annual specification changes.
  • ▹ AI latency: Streaming responses with partial rendering for immediate feedback.

🗺️ Future roadmap

Phase 2 includes LoR (Letter of Recommendation) management system with AI-assisted drafting, interview preparation chatbot using GPT-4, and a residency program matching algorithm. Integration with ERAS API for direct submission is planned for Q1 2026.

The USMLE Sarthi redesign successfully transformed a dated, cluttered website into a modern, AI-powered platform that empowers international medical graduates throughout their residency journey. By combining thoughtful UX with cutting-edge web technologies (Node.js, WebSocket, Redis), we delivered measurable improvements in user satisfaction, engagement, and career outcomes.
